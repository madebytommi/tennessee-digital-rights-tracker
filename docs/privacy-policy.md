# Project Privacy Policy

## Data minimization

The public website should use:

- No user accounts
- No comments database
- No first-party analytics
- No advertising
- No tracking pixels
- No embedded third-party social posts by default
- No forms collecting personal stories

## GitHub limitations

GitHub and GitHub Pages may still receive network and account information under their own policies. This project must not claim that visitors or contributors are anonymous.

## Contributions

Public GitHub activity may reveal contributor usernames, commit identities, issue activity, and timestamps.

Contributors who need greater privacy should use a pseudonymous GitHub account with an appropriate threat model. The project should not promise that this defeats platform, legal, or network-level identification.

## Sensitive submissions

Do not request sensitive information through GitHub issues.

For corrections involving personal or sensitive details, the project should publish a separate contact process after consulting an organization capable of handling that data responsibly.
