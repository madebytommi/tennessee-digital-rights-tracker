# Roadmap

## Phase 0 — Foundation

- [ ] Choose repository name and license
- [ ] Enable two-factor authentication
- [ ] Enable branch protection
- [ ] Publish methodology, editorial, privacy, and security policies
- [ ] Configure GitHub Pages
- [ ] Add validation workflow

## Phase 1 — Three-entry MVP

Publish three entries:

- [ ] Health-data privacy
- [ ] Government surveillance
- [ ] Online identity or age verification

Acceptance criteria:

- Every entry links to a primary source
- Fact and interpretation are separated
- Every uncertainty is stated
- Every entry includes a review date
- Every entry includes lawful action options
- No sensitive personal data is stored

## Phase 2 — Public contribution workflow

- [ ] Correction issue form
- [ ] New-entry proposal form
- [ ] Source-update form
- [ ] Pull-request review checklist
- [ ] Revision history convention

## Phase 3 — Better navigation

- [ ] Category filters
- [ ] Status filters
- [ ] Search
- [ ] Timeline view
- [ ] County and municipality tags
- [ ] Machine-readable index

## Phase 4 — Preservation

- [ ] Automated link checking
- [ ] Archive metadata
- [ ] File hashing
- [ ] Downloadable data export
- [ ] Mirror and backup process

## Not in the first version

- User accounts
- Anonymous tip submission
- Personal case tracking
- Discord integration
- AI-generated automatic publication
- Public database of individuals
- Real-time protest or organizer coordination
